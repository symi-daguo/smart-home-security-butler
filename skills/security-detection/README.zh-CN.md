# 安全检测引擎

智能家居 AI 安全管家的安全威胁检测引擎。

## 概述

安全检测引擎通过可配置的检测规则处理采集的智能家居数据，识别安全威胁、异常和潜在问题。

## 功能特性

- 多种检测类别（离家模式、设备故障、能耗、门禁、环境安全）
- 每条规则可配置严重等级
- 实时和周期性检测模式
- 异常检测的基线学习
- 告警生命周期管理（检测、确认、解决、静默）
- 误报追踪和自动阈值调整
- 按需安全扫描

## 检测类别

| 类别 | 默认严重等级 | 说明 |
|------|-------------|------|
| 离家模式异常 | 高 | 离家时的异常活动 |
| 设备故障 | 中 | 设备离线、低电量、通信错误 |
| 能耗异常 | 低 | 异常的功耗模式 |
| 门禁违规 | 高 | 非预期时间的门窗开启 |
| 环境危害 | 严重 | 烟雾、燃气、漏水检测 |

## 安装

此子技能已包含在智能家居 AI 安全管家中，无需单独安装。

## 配置

### 可选的环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DETECTION_INTERVAL` | 300秒 | 检测扫描间隔（秒） |
| `BASELINE_LEARNING_DAYS` | 14 | 基线学习周期（天） |

### 规则配置

每条检测规则可通过 API 配置：

```json
{
  "id": "device-fault",
  "enabled": true,
  "severity": "medium",
  "config": {
    "offlineTimeout": 300,
    "batteryWarning": 20,
    "batteryCritical": 10
  },
  "notification": {
    "enabled": true,
    "channels": ["telegram", "homeassistant"],
    "throttle": {
      "maxPerHour": 10,
      "cooldownSeconds": 300
    }
  }
}
```

## 严重等级

| 等级 | 通知方式 |
|------|---------|
| 严重 | 所有渠道，立即 |
| 高 | 主要渠道，立即 |
| 中 | 主要渠道，限流 |
| 低 | 仅汇总报告 |
| 信息 | 仅记录日志 |

## 告警生命周期

```
已检测 → 活动中 → 已确认 → 已解决 → 已关闭
              ↓
            已升级（如果未确认）
```

## 基线学习

- 默认学习周期：14 天
- 学习期间：观察模式，不发送通知
- 基线：时段模式、单实体范围、事件频率
- 进度每日追踪和报告
